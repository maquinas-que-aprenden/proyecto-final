El proyecto que abre puertas en regulacion IA

NormaBot es un sistema Agentic RAG que permite consultar en lenguaje natural el BOE y el EU AI Act, clasificar sistemas de IA por nivel de riesgo y generar checklists de cumplimiento deterministas. Apunta al sector que mas contrata IA: banca, seguros y consultoria.

Por que este proyecto, por que ahora

Desde febrero 2025, el EU AI Act prohibe practicas de riesgo inaceptable. En agosto 2025 entran obligaciones para modelos de proposito general. AESIA ya esta operativa. Las empresas necesitan urgentemente entender que regulaciones les afectan.

Modulos del Sistema

Un ReAct agent con 4 tools + pipeline ML + checklist determinista + MLOps completo + interfaz Streamlit. Cada modulo demuestra competencias diferentes del bootcamp.

📚
RAG Normativo (Corrective RAG)
ChromaDB · multilingual-e5-base · Ollama Qwen 2.5 3B · Score Fallback

Corrective RAG Pipeline

  • Retrieve: ChromaDB con intfloat/multilingual-e5-base → top-k chunks legislacion
  • Grade: Ollama Qwen 2.5 3B (local) evalua relevancia de cada documento
  • Score fallback: si Ollama no responde, fallback a puntuacion por similitud
  • Format context: documentos relevantes preparados para el orquestador
  • Generate: el orquestador (Bedrock Nova Lite) genera la respuesta final con citas

Chunking legal

  • Por articulo/seccion, no tokens arbitrarios
  • Metadata: ley, articulo, fecha, fuente
  • Embeddings: intfloat/multilingual-e5-base (CPU)
  • Modos de busqueda: base (semantica directa) y soft (priorizada por fuente)
⚠️
Clasificador de Riesgo (ML)
XGBoost · TF-IDF + SVD · spaCy · Grid Search · StratifiedKFold · SHAP · MLflow

Pipeline ML completo

  • Dataset: ~200-300 descripciones etiquetadas (real + sintetico fusionado)
  • Preprocesamiento: limpieza spaCy (es_core_news_sm), stopwords, lematizacion
  • Features: TF-IDF + SVD (reduccion dimensionalidad) + keywords de dominio manuales
  • Modelo: XGBoost con Grid Search + StratifiedKFold (k=5)
  • Tracking: MLflow registra cada experimento (3 datasets: real, artificial, fusionado)
  • Explicabilidad: SHAP features relevantes por prediccion
  • Override determinista: Anexo III EU AI Act (bypass ML para casos claros)
  • Reentrenamiento: pipeline incremental con augmented data (retrain.py)

4 clases EU AI Act

  • Inaceptable (prohibido)
  • Alto riesgo (obligaciones fuertes)
  • Riesgo limitado (transparencia)
  • Riesgo minimo (sin obligaciones)
📋
Checklist de Cumplimiento (Determinista)
Python puro · Sin LLM · 469 lineas · 23 tests

Generacion determinista (sin LLM)

  • Obligaciones: checklist hardcoded por nivel de riesgo EU AI Act
  • Recomendaciones SHAP: basadas en features relevantes de la prediccion ML
  • Deteccion borderline: alerta cuando la confianza del clasificador es baja
  • 100% testeable: 23 tests unitarios puros, sin dependencia de LLM
  • Disclaimer: "Informe preliminar generado por IA. Consulte profesional juridico."
🧠
Orquestador ReAct Agent
LangGraph · Amazon Bedrock Nova Lite · ContextVar Side-Channel · SQLite Memory

Arquitectura del agente

  • ReAct: create_react_agent de LangGraph con tool calling
  • LLM: Amazon Bedrock Nova Lite v1 (orquestacion + generacion de respuestas)
  • 4 tools: search_legal_docs, classify_risk, save_user_preference, get_user_preferences
  • Side-channel ContextVar: transporta citas y metadata verificadas fuera del LLM
  • SQLite checkpointer: persistencia de conversaciones (fallback a MemorySaver)
  • Pre-model hook: recorta historial a 30K tokens antes de cada llamada LLM
🔬
MLOps + Observabilidad
MLflow · DVC · GitHub Actions · Langfuse · RAGAS · CodeRabbit

Pipeline completo

  • MLflow: tracking experimentos clasificador (3 datasets), model registry
  • DVC: versionado corpus legal con S3 como remote
  • GitHub Actions: 5 workflows (pr_lint, ci-develop, cicd-main, eval, deploy-manual)
  • Langfuse: trazas de cada llamada LLM (degradacion graceful si no disponible)
  • RAGAS: evaluacion RAG automatica (context_precision, context_recall, faithfulness)
  • CodeRabbit: code review automatizado en PRs (logica, seguridad, rendimiento)
🖥️

Streamlit · Docker · AWS EC2

Riesgos y Mitigaciones

Riesgo alto

Alucinaciones en dominio legal

Un LLM que inventa articulos del BOE puede ser peligroso. Las citas deben ser exactas.

Mitigacion: Side-channel ContextVar — las citas vienen directamente de metadata ChromaDB, no del LLM. RAGAS evalua faithfulness automaticamente. Disclaimer visible obligatorio.
Riesgo alto

Complejidad del dominio legal

El texto legislativo es denso, con referencias cruzadas y terminologia especifica.

Mitigacion: Chunking por articulo, no por tokens. Metadata rica. Corrective RAG con grading de relevancia. Ollama local para evaluacion rapida.
Riesgo medio

Dataset clasificacion pequeño

200-300 ejemplos etiquetados es poco para ML clasico.

Mitigacion: Dataset fusionado (real + sintetico). StratifiedKFold. Override determinista Annex III para casos claros. Documentado como limitacion conocida.
Riesgo medio

Dependencia de Ollama local

El grading RAG depende de Ollama Qwen 2.5 3B corriendo en el servidor.

Mitigacion: Score-based fallback si Ollama no responde. Ollama instalado en container Docker con SHA256 verification. Entrypoint script lo inicia automaticamente.
Riesgo bajo

Latencia de respuesta

Multiples llamadas LLM (grading + orquestacion) pueden sumar latencia.

Mitigacion: Qwen 2.5 3B local es rapido para grading (~5 docs por query). Pre-model hook limita historial a 30K tokens. Score fallback elimina llamadas LLM si es necesario.
Riesgo bajo

Coordinacion 4 personas

4 tracks paralelos requieren buena comunicacion.

Mitigacion: PRs obligatorios con CodeRabbit review. CI/CD automatico. Modulos desacoplados con interfaces claras (@tool functions).
Limitaciones conocidas documentadas: (1) Dataset clasificacion pequeño (~200-300 ejemplos). (2) Corpus legal limitado a EU AI Act + BOE seleccionado. (3) Ollama requiere ~3GB RAM adicional en el servidor. (4) Sin NER especializado para entidades legales (spaCy solo para preprocesamiento). (5) Checklist determinista — no se adapta a casos no previstos.
