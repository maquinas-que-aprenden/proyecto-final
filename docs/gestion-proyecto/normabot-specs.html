Módulos del Sistema

Tres agentes LangGraph + NLP pipeline + MLOps completo + interfaz Streamlit. Cada módulo demuestra competencias diferentes del bootcamp.

📚
Agente RAG Normativo (Corrective RAG)
LangGraph · LangChain · ChromaDB · Groq API · Self-Reflection

Corrective RAG Pipeline

  • Retrieve: ChromaDB → top-k chunks legislación
  • Grade: LLM evalúa si chunks son relevantes
  • Transform query: reformula si no hay docs buenos
  • Web fallback: búsqueda web si sigue sin resultados
  • Generate: respuesta con citas exactas (artículo, sección)
  • Self-reflection: verifica respuesta soportada por fuentes

Chunking inteligente legal

  • Por artículo/sección, no tokens arbitrarios
  • Metadata: ley, artículo, fecha, BOE número
  • Embeddings: paraphrase-multilingual-MiniLM-L12-v2 (CPU)
⚠️
Agente Clasificador de Riesgo (ML)
scikit-learn · XGBoost · Grid Search · k-fold · MLflow · SHAP

Pipeline ML

  • Dataset: ~200-300 descripciones de sistemas etiquetadas manualmente
  • Features: TF-IDF + embeddings + features manuales (sector, tipo datos)
  • Modelos: XGBoost vs Random Forest vs LogReg
  • Validación: Stratified k-fold (k=5), Grid Search
  • Tracking: MLflow registra cada experimento
  • Explicabilidad: SHAP features relevantes

4 clases EU AI Act

  • Inaceptable (prohibido)
  • Alto riesgo (obligaciones fuertes)
  • Riesgo limitado (transparencia)
  • Riesgo mínimo (sin obligaciones)
📋
Agente de Informes de Cumplimiento
LLM (Groq/Gemini) · Templates · Markdown/PDF export

Estructura del informe generado

  • Resumen ejecutivo: qué normativa aplica y por qué
  • Clasificación de riesgo: nivel + justificación + artículos
  • Obligaciones: requisitos EU AI Act para ese nivel
  • Citas legales: artículos exactos BOE + EU AI Act
  • Recomendaciones: pasos concretos de cumplimiento
  • Disclaimer: "Informe preliminar, consulte profesional jurídico"
🔤
NLP Pipeline + Fine-tuning QLoRA
spaCy · HuggingFace · Unsloth · Llama 3.2 3B · Google Colab T4

NLP

  • NER de entidades legales españolas con spaCy (leyes, artículos, organismos)
  • Clasificación de artículos por tema con modelo fine-tuned

Fine-tuning QLoRA

  • Base model: Llama 3.2 3B
  • Framework: Unsloth (2-4x más rápido)
  • Hardware: Google Colab free T4 GPU
  • Task: clasificar artículos legales por tema/relevancia
  • Dataset: ~500 artículos etiquetados del BOE
🔬
MLOps + Observabilidad
MLflow · DVC · GitHub Actions · Langfuse · EvidentlyAI · RAGAS

Pipeline completo

  • MLflow: tracking experimentos clasificador + fine-tuning, model registry

  • GitHub Actions: CI/CD → test → lint → eval RAGAS → build → deploy HF
  • Langfuse: trazas de cada llamada LLM (5.000 trazas/mes gratis)
  • EvidentlyAI: dashboards de drift y calidad de datos

Riesgos y Mitigaciones

Riesgo alto

Complejidad del dominio legal

El texto legislativo es denso, con referencias cruzadas y terminología específica. El chunking puede romper la coherencia.

Mitigación: Chunking por artículo, no por tokens. Metadata rica. Self-reflection para verificar que la respuesta es coherente. Probar con abogados/juristas si es posible.
Riesgo alto

Alucinaciones en dominio legal

Un LLM que inventa artículos del BOE puede ser peligroso. Las citas deben ser exactas.

Mitigación: Corrective RAG con self-reflection. RAGAS evalúa faithfulness automáticamente. Disclaimer visible: "informe preliminar". Citas con link al BOE original.
Riesgo medio

Rate limiting Groq (14.400 req/día)

Si hay muchos usuarios simultáneos, Groq puede rate-limitear.

Mitigación: Fallback Groq → Gemini → Mistral free tier. Cache de respuestas frecuentes. Para demo es más que suficiente.
Riesgo medio

Dataset clasificación pequeño

200-300 ejemplos etiquetados manualmente es poco para ML clásico.

Mitigación: Augmentation con paráfrasis (LLM). class_weight='balanced'. El clasificador es complementario al contexto RAG. Documentar como limitación + proponer expansión.
Riesgo medio

Coordinación 4 personas

4 tracks paralelos requieren buena comunicación.

Mitigación: Schemas Pydantic día 1. Sync diario 15 min. PRs obligatorios. Día 10 checkpoint. Persona D es "glue" — su CI/CD integra el trabajo de todos.
Riesgo bajo

Fine-tuning en Colab free

La T4 gratuita puede desconectarse. QLoRA puede no converger.

Mitigación: Guardar checkpoints frecuentes en Drive. Si falla: usar modelo base sin fine-tuning (funciona, menos accuracy). QLoRA con Unsloth es muy eficiente en T4.
